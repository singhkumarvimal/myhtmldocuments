A Starter's Guide to Data Security Posture Management (DSPM) in Microsoft Purview

Learn how to discover, classify, and protect your sensitive data across your data estate.

1. What is Data Security Posture Management (DSPM)?

Data Security Posture Management (DSPM) is a security discipline focused on answering critical questions about your data: Where is our sensitive data? Who has access to it? How is it being used? And is it secure?

Unlike traditional security that focuses on perimeters and infrastructure, DSPM focuses on the data itself. It helps you automatically discover unknown data stores, classify sensitive data at scale, and identify security risks like overexposed data or insecure configurations.

Microsoft Purview is Microsoft's unified data governance platform that provides powerful tools to implement a DSPM strategy.

2. The Pillars of DSPM

A successful DSPM program is built on a continuous cycle of four key activities:

  1. Discover: Find all the places your data lives, including known databases, data lakes, and previously unknown "shadow data" stores.
  2. Classify: Scan the data to identify and label sensitive information (e.g., credit card numbers, personal ID numbers, health records) using classification rules.
  3. Protect: Remediate the risks you find. This could involve removing public access, tightening user permissions, or applying encryption.
  4. Monitor: Continuously monitor for new data stores, changes in permissions, and new security risks to maintain your security posture over time.

3. Step 1: Deploy a Microsoft Purview Account

Your first step is to create a Microsoft Purview account in your Azure subscription. This will be the central hub for all your data governance and DSPM activities.

  1. In the Azure Portal, search for and select "Microsoft Purview".
  2. Click + Create to start the deployment wizard.
  3. Select your Subscription and Resource Group.
  4. Provide a unique name for your Purview account.
  5. Configure networking. For initial testing, you can allow access from all networks, but for production, you should use private endpoints for security.
  6. Review and create the account. The deployment can take around 15-30 minutes.

Once deployed, open the Purview account and click on the "Open Microsoft Purview governance portal" button. This is where you will perform the next steps.

4. Step 2: Discover Your Data (Register & Scan Sources)

You can't protect what you don't know you have. The next step is to tell Purview where to look for your data.

  1. In the Purview governance portal, navigate to the Data Map.
  2. Select the Sources tab and click + Register.
  3. Choose the type of data source you want to scan. Let's use Azure Blob Storage as an example. Click "Continue".
  4. Select your Azure subscription and the specific storage account you want to register. Click "Register".
  5. Now that the source is registered, you need to scan it. Find your new source in the list, and click the "New Scan" icon.
  6. Configure the scan. You will need to provide credentials for Purview to access the data. The easiest and most secure method is to use a Managed Identity for your Purview account.
  7. Select a "scan rule set". The system default includes a wide range of classification rules for common sensitive data types (like credit cards, IBANs, etc.).
  8. Set a schedule for the scan (e.g., weekly) to keep the data map up to date.
  9. Save and run the scan.

5. Step 3: Classify Your Data (Review Scan Results)

Once the scan is complete, Purview will populate the Data Map with metadata and, most importantly, classifications.

  1. In the Purview portal, go to the Data Catalog and browse your assets.
  2. Find the storage account you scanned and drill down into the files.
  3. Select a file that has been scanned. The "Overview" tab will show you the "Classifications" that Purview automatically applied (e.g., "EU Debit Card Number").
  4. You can also manually add or remove classifications if needed.
  5. Use the Data Catalog search to find all assets with a specific classification, for example, search for "Credit Card Number" to see every file where Purview found that data type.

6. Step 4: Monitor Your Posture (Use Data Estate Insights)

DSPM is not a one-time project. Microsoft Purview provides a dashboard to monitor your posture continuously.

  1. In the Purview portal, navigate to Data Estate Insights.
  2. This dashboard gives you a bird's-eye view of your data estate.
  3. Asset Insights: See a summary of your data sources and the total number of discovered and classified assets.
  4. Classification Insights: Review the top classification categories found across your data. This helps you understand where your most sensitive data resides (e.g., you have 500 files with financial data and 200 with personal data).
  5. Glossary Insights: If you use the business glossary, you can see how well your data is documented.

Regularly review these insights to track the growth of your data estate, identify unscanned sources, and monitor trends in sensitive data discovery.

7. Best Practices for a Successful DSPM Program

  • Start Small, Then Expand: Don't try to boil the ocean. Start by scanning a few critical data sources, demonstrate value, and then expand your scope.
  • Automate Everything: Use scheduled scans and automated classification. Manual efforts do not scale.
  • Focus on Remediation: Discovery is only half the battle. Create a process for acting on the findings from Purview, such as working with data owners to fix overly permissive access.
  • Integrate with Your Security Operations: Forward alerts and insights from Purview to your SIEM (like Microsoft Sentinel) to correlate data risks with other security signals.
  • Educate Data Owners: DSPM is a shared responsibility. Use Purview's insights to educate business users about their data and empower them to be better data stewards.
